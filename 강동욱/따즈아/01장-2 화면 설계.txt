- 제약 사항 : 비용, 구제

기능성(기능제공)
- 적절성/적합성(목적 달성), 정밀성/정확성(정확,산출), 상호 운용성(다른 시스템), 보안성(접근권한), 호환성(기능)
신뢰성(오류 없이)
- 성숙성(고장 -> ), 고장 허용성(성능 유지), 회복성(복구)
사용성(다시 사용가능)
- 이해성(SW사용), 학습성(학습), 운용성(운용제어), 친밀성(다시 사용)
효율성
- 시간 효율성(시간), 자원 효율성(자원)
유지보수성(새로운 요구사항 -> 개선확장)
- 분석성(고장원인), 변경성(수정 쉬움), 안정성(최소화), 시험성(변경 검증)
이식성(다른환경)
- 적용성, 설치성, 대체성, 공존성(자원 공유)

UI프로토타입
개요 : 모형(테스트 가능), 간단(전체x), 핵심적 기능, 개선, 보안, 실제 사용자
장점 : 설득, 이해, 요구사항, 기능 불일치 저하, 개발 시간 감소, 사전에 오류 발견
단점 : 개선 및 보안(작업 시간 증가), 필요이상의 자원 소모, 중요한 작업 생략
종류
- 페이퍼 프로토타입 : 아날로그(스케치, 그림, 글), 개발 기간짧음, 비용 적음, 협의가 빠를때 사용
- 디지털 프로토타입 : 파워포인트, 전문가, 재사용가능
계획시 고려사항 : 개발목적, 환경, 일정(실제분석작업), 인원
작성시 고려사항 : 작성계획, 범위, 목표, 기간, 비용, 참조
제작 단계
- 1단계 : 분석, 요구확정
- 2단계 : 종이(그림), 편집도구, 핵심기능
- 3단계 : 사용자 확인, 추가, 수정, 의견
- 4단계 : 수정 후 합의, 보안작업

UI 시나리오 요건
- 완전성 : 상세하게 기술, 사용자중점
- 일관성
- 이해성 : 쉽게 이해가능해야한다.
- 가독성 : 읽기가 쉬워야한다
- 수정 용이성 : 수정 및 개선이 쉬워야한다
- 추적 용이성 : 변경된 요소에 대한 추적이 가능해야한다.
UI시나리오 문서로 인한 기대 효과(사용자의 요구사항 및 의사소통에 대한 오류 감소, 재작업 감소, 혼선 최소화, 불필요한 기능 감소, 소프트웨어 비용 감소, 개발속도 증가)

감성공학
개념 : 인체의 특징과 감정을 제품설계에 최대한 반영시키는 기술
관련 기술
- 생체 측정 기술 : 인간공학, 인지공학, 인간특성
- 인간의 오감 센서 및 감성 처리 기술 : 사용자인터페이스(실현기술)
- 감성 디자인 기술 : 산업디자인
- 새로운 감성을 창출하기 위한 기술 : 가상현실을 통한 인간에대한 적합성 판단
감성공학의 접근방법
- 감성공학 1류 : 인간의 감성 이미지를 측정하는 방법(제품 이미지 조사, 분석)
- 감성공학 2류 : 개인이 갖고 있는 이미지를 구체화하는 방법(개인성, 문화적 감성)
- 감성공학 3류 : 인간의 감각을 측정하고, 수학적 모델을 구축하여 활용
